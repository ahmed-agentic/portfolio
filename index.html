Case Study · 01 · NexusFlow
From a spinner to a
reasoning system.
Enterprise research teams were spending 3–5 hours per report manually synthesising information from dozens of sources. Quality was inconsistent. The process was not repeatable. The first attempt — a single GPT-4o call — produced plausible-looking but shallow output with no citations and no quality signal.
→
Built a LangGraph state machine with a dedicated Critic agent that scores output on four rubric dimensions and routes back to Researcher if quality drops below 0.72.
→
Added a Qdrant RAG layer that answered ~40% of Researcher sub-queries from the knowledge base, eliminating redundant API calls and cutting average cost from $0.18 to $0.11.
→
Streamed every agent thought event to the frontend via SSE — users can watch the system reason, reducing perceived wait time and building trust in the output.

Case Study · 06 · CliniqFlow
HIPAA compliance as
an engineering problem.
A healthcare network wanted to use EHR data to train models and generate operational insights. Legal blocked the project: patient data reaching LLM APIs required a Business Associate Agreement, which the vendor wouldn't sign. The challenge was making the data legally unidentifiable before it left the organisation's perimeter.
→
Applied HIPAA Safe Harbour: all 18 specified identifier types addressed via Presidio NLP, HMAC-SHA256 pseudonymisation, geographic generalisation, and date shifting.
→
Added k-anonymity validation (k=5) on quasi-identifier combinations before mart materialisation — a statistical guarantee against re-identification via combination attack.
→
Schema separation (raw / deidentified / staging / mart) enforced at the PostgreSQL GRANT level — no application-layer trust required for data isolation.
